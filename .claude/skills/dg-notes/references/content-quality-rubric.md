# Content Quality Rubric

Scoring rubric for evaluating knowledge note quality. Each dimension scored 1-5.

## Dimensions

### 1. Structure (1-5)
| Score | Criteria |
|-------|----------|
| 1 | No headings, wall of text |
| 2 | Some headings but inconsistent hierarchy |
| 3 | Clear H2 sections, logical flow |
| 4 | Well-organized with H2/H3 hierarchy, Related Topics section |
| 5 | Exemplary structure: overview → concepts → details → connections → references |

### 2. Clarity (1-5)
| Score | Criteria |
|-------|----------|
| 1 | Jargon-heavy, no definitions |
| 2 | Some explanation but assumes too much background |
| 3 | Clear for someone with domain familiarity |
| 4 | Accessible to curious generalists, key terms defined |
| 5 | Crystal clear at any level; complex ideas explained simply |

### 3. Completeness (1-5)
| Score | Criteria |
|-------|----------|
| 1 | Stub — title and a sentence |
| 2 | Basic definition, missing key aspects |
| 3 | Covers core concepts adequately |
| 4 | Comprehensive with examples and context |
| 5 | Authoritative reference; covers nuances, history, implications |

### 4. Connections (1-5)
| Score | Criteria |
|-------|----------|
| 1 | No wikilinks, isolated note |
| 2 | 1-2 links, no Related Topics section |
| 3 | 3-5 relevant links with Related Topics section |
| 4 | Rich linking with cross-domain connections |
| 5 | Hub note: well-connected, referenced by many, bridges domains |

## Frontmatter Compliance

Separate pass/fail checks:
- [ ] `title` present and descriptive
- [ ] `tags` present with domain-level tag
- [ ] `created` date in YYYY-MM-DD format
- [ ] `modified` date present and current
- [ ] `description` present (optional but recommended)
- [ ] No invalid or deprecated fields

## Overall Rating

| Average Score | Rating | Action |
|--------------|--------|--------|
| 4.0-5.0 | Excellent | No action needed |
| 3.0-3.9 | Good | Minor improvements suggested |
| 2.0-2.9 | Needs Work | Should be improved |
| 1.0-1.9 | Stub | Needs significant expansion or merge |
